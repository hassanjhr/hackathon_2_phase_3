# Todo Full-Stack Web Application

A secure, multi-user todo application built with Next.js 16+ (frontend) and FastAPI (backend), featuring JWT-based authentication and PostgreSQL database.

## Project Overview

This project demonstrates a production-ready full-stack web application following the Spec-Kit Plus workflow with complete spec-driven development. All code is generated by Claude Code specialized agents following constitutional principles.

## Architecture

- **Frontend**: Next.js 16+ with App Router, TypeScript, TailwindCSS, Better Auth
- **Backend**: FastAPI with SQLModel, JWT authentication, asyncpg
- **Database**: Neon Serverless PostgreSQL
- **Authentication**: JWT tokens with Better Auth integration

## Features

- ✅ User registration and authentication
- ✅ Secure JWT-based session management
- ✅ Create, read, update, delete tasks
- ✅ Toggle task completion status
- ✅ User data isolation (users can only access their own tasks)
- ✅ Responsive UI (desktop and mobile)
- ✅ RESTful API with proper HTTP status codes

## Quick Start

### Prerequisites

- Node.js 18+
- Python 3.11+
- Neon PostgreSQL account

### Setup

1. **Clone and navigate to the project**:
```bash
git clone <repository-url>
cd hackathon_2_phase_2
```

2. **Configure environment variables**:
```bash
cp .env.example .env
# Edit .env with your Neon database URL and generate a JWT secret
```

3. **Backend setup**:
```bash
cd backend
python3 -m venv venv
source venv/bin/activate  # On Windows: venv\Scripts\activate
pip install -r requirements.txt
cp .env.example .env
# Edit backend/.env with your configuration
uvicorn src.main:app --reload --host 0.0.0.0 --port 8000
```

4. **Frontend setup** (in a new terminal):
```bash
cd frontend
npm install
cp .env.local.example .env.local
# Edit frontend/.env.local with your configuration
npm run dev
```

5. **Access the application**:
- Frontend: http://localhost:3000
- Backend API: http://localhost:8000
- API Docs: http://localhost:8000/docs

## Security

- JWT tokens for stateless authentication
- Password hashing with bcrypt
- User data isolation at database level
- CORS configuration for frontend-backend communication
- Environment-based secrets management

## Testing

- Backend: `pytest` for API endpoint testing
- Frontend: Manual testing for authentication and CRUD flows
- Integration: End-to-end authentication and data isolation testing

## Constitutional Principles

This project adheres to the following principles:

1. **Spec-First Development** - All features defined before implementation
2. **Security by Default** - JWT authentication on all endpoints
3. **User Data Isolation** - Database-level filtering by user_id
4. **Reproducibility** - Complete documentation trail
5. **Automation-First** - All code generated by Claude Code agents
6. **Production Realism** - Real technologies (Neon, Better Auth, FastAPI, Next.js)
